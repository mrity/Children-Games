## 一、代码架构优化（高优先级）

### 1.1 模块化拆分 app.js ⭐⭐⭐
**现状问题**：
- app.js 有 1,789 行，包含题库数据、游戏逻辑、动画控制、存储管理等多个职责
- createQuestionTypes() 函数嵌套过深，混合了数据生成和业务逻辑
- 代码职责不清晰，维护困难

**优化方案**：
```
js/
├── core/
│   ├── constants.js ✅ (已完成 - 配置常量)
│   ├── state.js ✅ (已完成 - 状态管理)
│   └── questions.js (新增 - 题库数据)
├── game/
│   ├── wheel.js (摩天轮动画逻辑)
│   ├── answer.js (答题系统)
│   └── hint.js (提示/讲解系统)
├── ui/
│   ├── navigation.js (页面导航)
│   └── render.js (渲染函数)
└── storage/
    └── progress.js (进度管理)
```

**实施步骤**：
1. ✅ 已创建目录结构和基础模块（constants.js, state.js）
2. 将题库数据抽离到 questions.js
3. 拆分游戏逻辑到各个模块
4. 修改 index.html 使用 ES6 模块导入
5. 全面测试确保功能正常

**收益**：
- 提升可维护性，便于多人协作
- 减少合并冲突
- 代码职责清晰，易于扩展

---

### 1.2 题库数据外部化 ⭐⭐
**现状问题**：
- 60 道题硬编码在 app.js 中
- 扩展题库需要修改代码
- 非技术人员无法直接编辑题库

**优化方案**：
```javascript
// data/questions.json
{
  "version": "1.0",
  "zones": [
    {
      "id": "observe",
      "name": "观察小舞台",
      "types": [...]
    }
  ]
}

// 异步加载题库
async function loadQuestions() {
  const response = await fetch('data/questions.json');
  return await response.json();
}
```

**收益**：
- 非技术人员可直接编辑题库
- 支持题库版本管理
- 便于动态扩展题型和题目数量
