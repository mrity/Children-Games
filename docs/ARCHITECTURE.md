# 摩天轮乐园 - 架构设计文档

> 最后更新：2026-04-16

## 项目概述

摩天轮乐园是一个面向儿童的逻辑启蒙网页游戏，通过摩天轮场景帮助儿童学习顺序观察和位置推理。

### 技术栈

- **前端框架**：纯 HTML + CSS + JavaScript（ES6+）
- **构建工具**：Vite 5.x
- **代码规范**：ESLint + Prettier
- **版本控制**：Git
- **部署方式**：GitHub Pages

### 核心特性

- 无依赖，轻量级
- 响应式设计，支持移动端
- 本地存储进度，支持导入导出
- 完整的可访问性支持
- 分步提示和讲解系统

### 模块划分

#### 1. 核心模块（js/core/）

**constants.js** - 配置常量
- 存储键名
- 动画时长配置
- 摩天轮结构常量
- 星级评分阈值
- 动物配置

**state.js** - 全局状态管理
- DOM 缓存对象
- 游戏状态
- 动画状态
- 进度数据

**questions.js** - 题库数据（规划中）
- 题目生成逻辑
- 题型配置
- 答案验证

#### 2. 游戏模块（js/game/）

**wheel.js** - 摩天轮动画逻辑（规划中）
- 转动动画控制
- 座舱位置计算
- 动画状态管理

**answer.js** - 答题系统（规划中）
- 答案提交
- 正确性判断
- 星级计算

**hint.js** - 提示/讲解系统（规划中）
- 提示显示
- 分步讲解
- 回放控制

#### 3. UI 模块（js/ui/）

**navigation.js** - 页面导航（规划中）
- 页面切换
- 导航状态管理

**render.js** - 渲染函数（规划中）
- 场景渲染
- DOM 更新
- 增量渲染优化

#### 4. 存储模块（js/storage/）

**progress.js** - 进度管理（规划中）
- 进度加载/保存
- 数据验证
- 导入/导出

```javascript
  answer: "位置1",             // 正确答案
  hints: [...],                // 提示列表

```

## 核心流程

### 游戏初始化流程

```
1. 加载 DOM 元素
2. 检查浏览器兼容性
3. 初始化存储系统（localStorage 或内存降级）
4. 加载游戏进度
5. 生成题库数据
6. 绑定事件监听器
7. 渲染首页
```

### 答题流程

```
1. 用户选择答案
2. 验证答案正确性
3. 更新 UI 反馈（正确/错误）
4. 计算星级（基于错误次数）
5. 更新进度数据
6. 保存到本地存储
7. 自动进入下一关（如果答对）
```

### 摩天轮转动流程

```
1. 用户设置转动参数（方向、圈数、格数）
2. 计算总转动步数
3. 开始动画循环：
   a. 执行单步转动动画
   b. 更新场景数据
   c. 重新渲染场景
   d. 继续下一步（如果未完成）
4. 转动完成，恢复交互
```

## 性能优化

### 已实施的优化

1. **DOM 渲染优化**
   - 增量更新：仅更新变化的元素
   - 避免全量重绘
   - 使用 DocumentFragment 批量插入

2. **动画性能优化**
   - 优先使用 Web Animations API
   - 降级使用 requestAnimationFrame
   - 避免使用 setTimeout 轮询

3. **事件处理优化**
   - 使用事件委托减少监听器数量
   - 防抖和节流（窗口 resize）

4. **移动端优化**
   - 触摸目标最小 44x44px
   - 硬件加速（transform）
   - 触摸反馈优化

### 待优化项

1. 图片资源压缩
2. CSS 代码分割和压缩
3. 代码懒加载
4. Service Worker 缓存

## 技术决策

### 为什么选择纯 JavaScript？

- **轻量级**：无框架依赖，加载快
- **易维护**：代码简单直观
- **易部署**：无需构建步骤（开发阶段）
- **学习成本低**：适合教育项目

### 为什么使用 Vite？

- **开发体验好**：快速热更新
- **构建速度快**：基于 esbuild
- **配置简单**：零配置即可使用
- **现代化**：支持 ES 模块

### 为什么使用 localStorage？

- **简单**：无需后端服务器
- **即时**：数据立即可用
- **隐私**：数据保存在本地
- **降级方案**：内存存储兜底
