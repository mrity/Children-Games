## 实施策略

### 阶段 1：添加 JSDoc 类型注释（优先级最高）

**目标**：为所有核心函数添加完整的 JSDoc 注释

**范围**：
- 导航函数（navigateTo, renderAll 等）
- 游戏逻辑函数（submitAnswer, computeStars, advanceScene 等）
- 进度管理函数（loadProgress, saveProgress, exportProgress 等）
- 渲染函数（renderScene, renderGame, renderProgress 等）
- 工具函数（animalSlot, rotationStepsForAnimalToSlot 等）

**注释格式**：
```javascript
/**
 * 函数描述
 * @param {类型} 参数名 - 参数说明
 * @returns {类型} 返回值说明
 * @throws {Error} 异常说明（如果有）
 */
```

**预计工作量**：约 50-60 个函数需要添加注释

---

### 阶段 2：题库数据外部化（中等优先级）

**目标**：将题库数据从 app.js 中提取到独立的 JSON 文件

**步骤**：
1. 创建 `data/questions.json` 文件
2. 将 createQuestionTypes() 生成的数据导出为 JSON
3. 在 app.js 中添加异步加载函数
4. 修改初始化流程，等待题库加载完成
5. 添加加载失败的错误处理

**收益**：
- 非技术人员可直接编辑题库
- 支持题库版本管理
- 便于动态扩展题型

---

### 阶段 3：完成模块化拆分（低优先级，工作量大）

**目标**：将 app.js（2100+ 行）拆分为多个模块

**模块划分**：
```
js/
├── core/
│   ├── constants.js ✅ (已完成)
│   ├── state.js ✅ (已完成)
│   ├── sound.js ✅ (已完成)
│   └── questions.js (新增 - 题库数据)
├── game/
│   ├── wheel.js (摩天轮动画逻辑)
│   ├── answer.js (答题系统)
│   └── hint.js (提示/讲解系统)
├── ui/
│   ├── navigation.js (页面导航)
│   └── render.js (渲染函数)
└── storage/
    └── progress.js (进度管理)
```

**注意事项**：
- 需要修改 index.html 使用 ES6 模块导入
- 需要处理模块间的依赖关系
- 需要全面测试确保功能正常
- 这是最大的重构任务，建议分步实施
